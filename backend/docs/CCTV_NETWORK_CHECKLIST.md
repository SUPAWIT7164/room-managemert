# Checklist: ตรวจ .env และเครือข่ายกล้อง (เมื่อ count-people ได้ 503)

เมื่อหน้า CCTV ได้ **503** และข้อความ "กล้องหรือเซิร์ฟเวอร์ภาพไม่ให้บริการชั่วคราว (503)" ให้ทำตามขั้นตอนด้านล่างเพื่อตรวจ **.env** และ **เครือข่ายจากเซิร์ฟเวอร์ bms-dev ไปกล้อง**

---

## 1. ตรวจ .env บนเซิร์ฟเวอร์ bms-dev

1. เข้าเซิร์ฟเวอร์ที่รัน backend (bms-dev.lanna.co.th หรือเครื่องที่ deploy)
2. เปิดไฟล์ `.env` ในโฟลเดอร์ backend
3. ตรวจค่า:
   - **CCTV_BASE_URL** ต้องชี้ไปที่ **IP กล้องโดยตรง** เช่น `http://192.168.24.1`
   - **ห้าม** ชี้ไปที่ URL ของ bms-dev (เช่น `https://bms-dev.lanna.co.th`) หรือ proxy อื่น
   - **CCTV_USERNAME** และ **CCTV_PASSWORD** ต้องตรงกับที่ตั้งไว้บนกล้อง

ตัวอย่างที่ถูก:
```env
CCTV_BASE_URL=http://192.168.24.1
CCTV_USERNAME=admin
CCTV_PASSWORD=L@nnac0m
```

---

## 2. รันสคริปต์ทดสอบบนเซิร์ฟเวอร์ bms-dev

จากโฟลเดอร์ backend บนเซิร์ฟเวอร์ รัน:

```bash
node scripts/test-cctv-snapshot.js
```

- **ถ้าได้:** `✅ ดึงภาพสำเร็จ: XXXXX bytes` และมีไฟล์ `test-snapshot.jpg`  
  → เซิร์ฟเวอร์เข้าเครือข่ายกล้องได้ ปัญหา 503 อาจมาจากจังหวะ/โหลด — ลอง refresh หน้า CCTV อีกครั้ง

- **ถ้าได้ 503 / Timeout / Connection Refused:**  
  → สคริปต์จะพิมพ์ **ขั้นต่อไป** ออกมา (เช่น ตรวจ .env, VPN, LAN) — ทำตามนั้น

---

## 3. ใช้ endpoint diagnose (ไม่ต้อง SSH)

เปิดเบราว์เซอร์ **หลัง login** แล้วเข้า:

```
https://bms-dev.lanna.co.th/api/cctv/diagnose
```

- **ok: true** → เซิร์ฟเวอร์ดึงภาพจากกล้องได้ ปัญหาอาจอยู่ที่จังหวะ/โหลด
- **ok: false** → จะมี **message**, **hint**, **nextSteps** บอกว่าต้องตรวจอะไร (เช่น .env, VPN, LAN)

---

## 4. ถ้าเซิร์ฟเวอร์เข้า 192.168.24.1 ไม่ได้ (เครือข่ายคนละวง)

**192.168.24.1** เป็น IP ภายใน (LAN) — ถ้าเซิร์ฟเวอร์ bms-dev อยู่บน **cloud / อินเทอร์เน็ต** จะเข้า IP นี้โดยตรงไม่ได้

ทางเลือก:

1. **VPN**  
   - ให้เซิร์ฟเวอร์ bms-dev เชื่อม VPN เข้าเครือข่ายเดียวกับกล้อง แล้วใช้ IP กล้องใน VPN (หรือ 192.168.24.1 ถ้า VPN route ไปได้)

2. **ย้าย backend ไปรันในเครื่องที่อยู่ LAN เดียวกับกล้อง**  
   - เช่น รัน Node backend บนเครื่องในออฟฟิศที่อยู่เครือข่าย 192.168.24.x แล้วให้ frontend (bms-dev) เรียก API ของเครื่องนั้น

3. **Port forward / reverse proxy**  
   - ถ้ามี gateway ใน LAN ที่เข้าถึงกล้องได้ อาจตั้ง reverse proxy หรือ port forward ให้เซิร์ฟเวอร์ bms-dev เรียก gateway แทนการเรียก 192.168.24.1 โดยตรง (ต้องออกแบบความปลอดภัยให้เหมาะสม)

---
